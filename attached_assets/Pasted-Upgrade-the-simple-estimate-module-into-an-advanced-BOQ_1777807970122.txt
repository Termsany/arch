Upgrade the simple estimate module into an advanced BOQ and costing system.

Goal:
Offices can manage BOQ categories, item library, material cost, labor cost, waste percentage, profit margin, and final pricing.

Tasks:

1. Database
Add table:

boq_categories:
- id
- office_id
- name
- description
- sort_order
- created_at
- updated_at

Add table:

boq_items_library:
- id
- office_id
- category_id
- item_name
- default_unit
- default_material_cost
- default_labor_cost
- default_waste_percentage
- default_profit_margin
- notes
- created_at
- updated_at

Update project_estimates or add missing columns:
- office_id
- category_id
- material_unit_cost
- labor_unit_cost
- waste_percentage
- profit_margin
- unit_cost_before_profit
- total_cost_before_profit
- total_price

Calculations:
unit_cost_before_profit = material_unit_cost + labor_unit_cost
total_cost_before_profit = quantity * unit_cost_before_profit * (1 + waste_percentage / 100)
total_price = total_cost_before_profit * (1 + profit_margin / 100)

2. Backend API
BOQ categories:
- GET /api/boq/categories
- POST /api/boq/categories
- PUT /api/boq/categories/:id
- DELETE /api/boq/categories/:id

BOQ library:
- GET /api/boq/library
- POST /api/boq/library
- PUT /api/boq/library/:id
- DELETE /api/boq/library/:id

Update estimate endpoints:
- Support category_id
- Support material/labor/waste/profit fields
- Return calculated totals
